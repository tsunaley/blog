使用MkDocs搭建个人博客

时间：2019年1月3日

一些想说的

咸鱼超终于搭好了她的博客之后表示十分开心，想着往博客里放点什么东西
（其实是遭受了一下午高数摧残之后不想背军理了）
所以咸鱼超就把搭博客的过程写写，免得以后忘了

这个博客是使用的MkDocs搭建的
网上也有很多相关的教程，但是我在搭建的过程中仍然遇到了很多问题
所以这里会主要写我的搭建过程

环境准备

首先需要安装python
在Ubuntu下打开终端输入 

apt install python

然后安装pip 

apt install python-pip

安装MkDocs 

pip install mkdocs

检查MkDocs是否安装成功 

mkdocs

如果出现一些MkDocs的相关信息，就说明安装好了

然后是git
博客嘛，肯定得给人看，所以我们要用github pages
安装git 

apt-get install git

检查git是否安装成功 

git

如果出现一些git的相关信息，就说明安装好了
我们进入下一步

下一步

（好吧，我实在不知道该怎么起名了）
因为要使用github，所以
1. 先注册一个GitHub账户 github
2. 创建一个公共仓库
3. 进入仓库，获得仓库地址https://github.com/xxxxx/emmm

然后我们在本地克隆一份
进入你想要的地址，输入

git clone https://github.com/xxxxx/emmm

进入仓库
创建MkDocs项目
输入

mkdocs new dei

即可创建
进入dei
执行

mkdocs serve

他会给你一个网址http://127.0.0.1:8000
在浏览器访问它，就可以看到默认主页了
现在大体上已经成了

细节

（对，我就是个起名废）
写博客只需要将.md文件放在docs/目录中就行了
文件需要写成markdown，关于markdown的语法

MkDocs很多细节都是在mkdocs.yml中设置的，这里介绍几个
更多的参考官方文档Configuration-MkDcos

进入mkdocs.yml

site_name

site_name: Tsunaley's blog

这个是作为主标题

theme

theme: 'windmill'

这个是用于设置主题，我使用的就是windmill（第三方主题需要安装）
具体可以看 Custom themes - MkDocs

copyright

copyright: Tsunaley All Rights Reserved.

这个是写在页脚的

nav

nav: 
- 主页: 'index.md'
- 各种: 
    - 新年快乐: 'xxxx.md'

这个是用于设置导航

部署

在终端执行

mkdocs gh-deploy

它会自动生成，期间会提示你输入账户名和密码，按提示来就好
如果报错可以试试

mkdocs gh-deploy --force

然后在GitHub的相应仓库的setting里的github pages能找到对应网页
